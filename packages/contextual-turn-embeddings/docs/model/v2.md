# v2 — Registro de divergencias respecto de BERT

El **v2** (`model/v2.py`, clase `ContextualTurnModelV2`) es un **port lo más fiel
posible** de la implementación de BERT
([`modeling_bert.py`](https://github.com/huggingface/transformers/blob/main/src/transformers/models/bert/modeling_bert.py))
aplicada **sobre turnos de diálogo** en vez de tokens. Sirve como **baseline estándar** contra el
cual comparar el v1 (`model/v1.py`, un Transformer custom). El v1 queda **intacto**.

## En simple (TL;DR)

**v2 es BERT, pero corriendo sobre *turnos de diálogo* en lugar de palabras.** Copiamos la
implementación de BERT casi tal cual; lo único que cambiamos es lo que **obliga** el hecho de que un
turno no es una palabra:

| | BERT (palabras) | v2 (turnos) |
|---|---|---|
| **Entrada** | un ID de palabra que busca en una tabla | el **vector del turno** directo (`e_t` de Dialog2Flow) — no hay "diccionario de turnos" |
| **Marca extra** | si la palabra es de la frase A o B | **quién habla** (usuario / sistema) |
| **Salida** | qué palabra sigue (probabilidad sobre el vocabulario) | el **embedding** del turno objetivo *(por ahora; el "probabilidad sobre vocabulario" llega con el codebook)* |
| **Mirar el futuro** | sí (bidireccional) | sí en Bidi; en AR le ponemos una **máscara** para que mire solo el pasado |

**Todo lo demás —la atención, las capas, las normalizaciones, la inicialización— es BERT idéntico.**
Las tablas de abajo son el **detalle fino** (para auditar línea por línea); con leer esto ya se
entiende la idea.

> **Principio:** lo que no hace falta cambiar, se deja igual; **toda** divergencia se registra acá.
> En el código, cada línea está anotada con `# FIEL` o `# DIFF`.
>
> 📑 **Índice rápido con `archivo:línea`:** [`v2_diff_recap.md`](v2_diff_recap.md).

## Jerarquía OOP (espeja la de BERT)

```
BertTurnPreTrainedModel        ↔  BertPreTrainedModel   (_init_weights)
  └─ BertTurnModel             ↔  BertModel             (embeddings + encoder, sin pooler)
ContextualTurnModelV2          ↔  BertForMaskedLM       (wrapper: self.bert + heads)
```

## Lo que se mantiene FIEL a BERT

- **Post-LayerNorm**: en `BertTurnSelfOutput` y `BertTurnOutput`, `LayerNorm(h + input)` después del
  residual (BERT es post-LN). *(El v1 era **pre-LN** — ver abajo.)*
- **Atención**: escala `attention_head_size ** -0.5`, máscara **aditiva**, softmax, dropout sobre los
  pesos de atención (camino *eager*).
- **Activación** GELU (erf), vía el mismo patrón `ACT2FN[config.hidden_act]`.
- **Orden de operaciones** de `BertLayer`: attention → intermediate → output(intermediate, attention).
- **`BertPredictionHeadTransform`**: `Dense → GELU → LayerNorm`, idéntico.
- **`layer_norm_eps = 1e-12`** y **init** `normal_(0, 0.02)` para Linear/Embedding, LayerNorm a (1, 0)
  (réplica de `BertPreTrainedModel._init_weights`).

## Registro de divergencias

### A. Obligatorias (turno ≠ token)

| # | Qué | BERT | v2 | Por qué | Dónde |
|---|---|---|---|---|---|
| 1 | Entrada | `word_embeddings(input_ids)` (lookup discreto) | `e_t` **continuo** + `input_proj` (`Linear` si `input_dim≠hidden`) | los turnos no son tokens de un vocab | `BertTurnEmbeddings` |
| 2 | `token_type_embeddings` | segmento A/B | **`speaker_embeddings`** (mismo mecanismo aditivo) | rol del hablante en vez de segmento | `BertTurnEmbeddings` |
| 3 | `position_embeddings` | aprendidas, `max_position=512` | aprendidas, `max_turns=64` | mismo mecanismo, otra escala | `BertTurnEmbeddings` |
| 4 | Vocab / decoder | `Linear(hidden, vocab)` **atado** a `word_embeddings` | **Fase 1:** decoder continuo `Linear(hidden, input_dim)` (predice un embedding). **Fase 2:** `head_mode` = `tied_continuous` (`H@E_D2F.T`) / `tied_codebook` / `learned` | no hay vocabulario de turnos discreto | `BertTurnPredictionHead` |
| 5 | `pooler` + NSP | presentes | **omitidos** | no se usan | `ContextualTurnModelV2` |
| 6 | Objetivo | MLM (cross-entropy sobre vocab) | masked-turn recon (MSE+cos) / contrastivo (Fase 1) → clasificación sobre codebook (Fase 2) | salida continua, no vocab | `train.compute_objectives` (sin cambios) |
| 7 | Máscara causal | BERT es **solo bidireccional** | máscara causal **aditiva opcional** si `attention_mode='autoregressive'` (default `bidirectional` = MLM fiel) | conservar la variante AR del v1 | `ContextualTurnModelV2._extended_attention_mask` |

### B. De *plumbing* (no arquitectónicas)

| Qué | BERT | v2 | Nota |
|---|---|---|---|
| Despacho de atención | `ALL_ATTENTION_FUNCTIONS` (eager/sdpa/flash) | **solo eager** (`eager_attention_forward` inline) | misma matemática |
| `BertLayer` extras | cross-attention, `apply_chunking_to_forward`, `GradientCheckpointingLayer`, KV-cache | **omitidos** | no cambian la arquitectura encoder-only |
| Config | `BertConfig` | shim `_BertTurnConfig` (deriva nombres estilo-BERT del `ModelConfig`) | para que cada `BertTurn*` quede *verbatim* |
| Persistencia | `PreTrainedModel.from_pretrained` | formato propio (`config.json` + `model.safetensors`), igual que el v1 | interoperable con el resto del paquete |

## Diferencias v1 ↔ v2 (las dos versiones nuestras)

| Aspecto | v1 (`ContextualTurnModel`) | v2 (`ContextualTurnModelV2`) |
|---|---|---|
| LayerNorm | **pre-LN** (`nn.TransformerEncoder(norm_first=True)`) | **post-LN** (BERT) |
| Salida | `h_t = LayerNorm(e_t + Δ)` (`output_residual`) | `h_t = ` salida del encoder (sin residual a base) |
| `layer_norm_eps` | `1e-5` (default de PyTorch) | `1e-12` (BERT) |
| Implementación | `nn.TransformerEncoder` | módulos `BertTurn*` porteados |
| Init | defaults de PyTorch | init de BERT (`std=0.02`) |

Estas diferencias son **el objeto de estudio**: comparar v1 vs v2 (mismo f1, mismo held-out, mismo
objetivo) aísla el efecto de la arquitectura BERT-fiel frente a la custom.

## Uso

```python
from contextual_turn_embeddings import ModelConfig, build_model

cfg = ModelConfig(arch="v2", input_dim=768, hidden_dim=768, num_layers=6,
                  num_heads=8, max_turns=64, attention_mode="bidirectional")
model = build_model(cfg)        # -> ContextualTurnModelV2 (arch="v1" -> ContextualTurnModel)
```

`ContextualTurnModelV2` es **drop-in** en `train.compute_objectives` y `encode.encode_dialogues`
(misma firma `forward` + `mask_embedding`/`reconstruction_head`/`next_turn_head`/`speaker_embedding`).
